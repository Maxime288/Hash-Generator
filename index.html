Les algorithmes

Comprendre les différences entre les fonctions de hachage cryptographique.

MD5 1992

Message Digest 5

Conçu par Ron Rivest, MD5 produit un hash de 128 bits (32 caractères hexadécimaux). Il a été largement utilisé pour la vérification d'intégrité de fichiers.

Aujourd'hui considéré comme cryptographiquement cassé : des collisions peuvent être générées en quelques secondes sur un ordinateur ordinaire.

Collisions connues Ne pas utiliser
Encore acceptable pour : checksums non-critiques, identifiants de cache
SHA-1 1995

Secure Hash Algorithm 1

Conçu par la NSA, SHA-1 produit un hash de 160 bits (40 caractères hexadécimaux). Il a longtemps été la référence pour les signatures numériques.

Officiellement déprécié depuis 2017 suite à l'attaque SHAttered de Google, qui a démontré la première collision pratique.

Déprécié Collision démontrée
Retiré de : navigateurs, certificats SSL/TLS, Git (en migration)
SHA-256 2001

Secure Hash Algorithm 256

Membre de la famille SHA-2, SHA-256 produit un hash de 256 bits (64 caractères hexadécimaux). C'est le standard actuel de référence en cryptographie.

Aucune collision connue. Résistant aux attaques actuelles. Utilisé partout dans les systèmes modernes.

Standard recommandé Aucune collision
Utilisé dans : HTTPS, Bitcoin, JWT, signatures numériques, Git
SHA-512 2001

Secure Hash Algorithm 512

Aussi membre de SHA-2, SHA-512 produit un hash de 512 bits (128 caractères hexadécimaux). Plus lent que SHA-256 mais avec une marge de sécurité plus grande.

Particulièrement performant sur les processeurs 64 bits. Recommandé pour les applications nécessitant une sécurité maximale.

Haute sécurité 512 bits
Utilisé pour : signatures de fichiers volumineux, authentification serveur
Algorithme Taille du hash Vitesse Sécurité Usage recommandé
